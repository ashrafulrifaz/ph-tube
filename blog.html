QUESTION 1: Discuss the scope of var, let, and const

ANSWER: We can compare scope to room. We can call the whole house global scope and each room of the house can be called block or functional scope. JavaScript has 3 variables var, let and const. Var is the global scope. We can access the value declared with var from anywhere in the code. Even from inside of a function. Beside, Let and Const is block scope. If we declared a value using let outside of a function or block scope, we can access it inside a function or block scope. But we can't redeclared or change a global let variable inside a function or block Scope. As well as Const is a block scope. We can't access a const variable inside or outside of a block scope.

QUESTION 2: Tell us the use cases of null and undefined

ANSWER: We use null to check the absence of a value. Also for represent an empty value and represent a missing or unknown value.
Beside we use undefined to represent the unintentional absence or missing of a value because if we declare a variable but do not assign a value, the variable will be undefined.

QUESTION 3: What do you mean by REST API?

ANSWER: An API, or application programming interface, is a set of rules that define how applications or devices can connect to and communicate with each other. A REST API is an API that conforms to the design principles of the REST, or representational state transfer architectural style. For this reason, REST APIs are sometimes referred to RESTful APIs.REST APIs communicate via HTTP requests to perform standard database functions like creating, reading, updating, and deleting records. For example, a REST API would use a GET request to retrieve a record, a POST request to create one, a PUT request to update a record, and a DELETE request to delete one.
